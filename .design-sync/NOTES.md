# design-sync notes — Decimal (app-shaped repo)

- This is an APP, not a packaged DS. Bundle entry is `frontend/src/ds-entry.ts` (must live inside
  `frontend/` — the converter walks up from `--entry` to the nearest named package.json).
  Build command: `node .ds-sync/package-build.mjs --config .design-sync/config.json
  --node-modules frontend/node_modules --entry frontend/src/ds-entry.ts --out ./ds-bundle`.
- `cssEntry` must be a COMPILED stylesheet (the converter appends its raw text; relative @imports
  break). `frontend/src/styles/decimal/ds-sync-entry.css` is GENERATED by `.design-sync/build-css.sh`
  (wired as cfg.buildCmd) from tokens.css + components.css + pages.css. Re-run before any re-sync
  if the decimal CSS changed. cssEntry paths are package-bounded (must be under frontend/).
- Every preview MUST wrap in `<div className="dec">` — all CSS is scoped under `.dec`.
- Config paths (tsconfig, cssEntry, componentSrcMap) are PACKAGE-relative (frontend/), not repo-relative.

## Known render warns
- `[FONT_REMOTE]` Bricolage Grotesque / JetBrains Mono via Google Fonts @import — expected, loads at runtime.
- Geist is host-served → `runtimeFontPrefixes: ["Geist"]` suppresses the missing-font warn.

## Re-sync risks
- `ds-sync-entry.css` is a generated concatenation — silently stale if buildCmd isn't run after CSS edits
  (buildCmd is wired, but a manual converter invocation bypasses it; use resync.mjs).
- `conventions.md` enumerates the class vocabulary — re-validate its names against `_ds_bundle.css`
  whenever styles/decimal/*.css classes change (grep each named class).
- Dark theme SHIPS inside tokens.css ([data-theme="dark"] block) — activation is the data-theme
  attribute on an ancestor. app-dark.css is legacy/non-decimal surfaces only; not needed.
- Only 5 React components exist; the real system is the class vocabulary — new app components
  (e.g. future flow-builder canvas parts) should be added to ds-entry.ts + componentSrcMap + previews.
